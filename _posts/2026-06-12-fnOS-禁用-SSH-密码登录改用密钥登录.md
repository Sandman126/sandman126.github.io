为 fnOS 禁用 SSH 密码登录（改用密钥登录）能有效防止暴力破解，大幅提升系统安全性。

操作可以分为两个主要步骤：**先在 Web 管理界面启用 SSH 和用户权限**，再到**终端里修改配置文件禁用密码**。

### 📝 步骤详解

#### 1. 开启 SSH 服务与用户权限
从 fnOS 系统版本 `v1.1.19` 开始，为提升安全性，系统**默认关闭了 SSH 服务和相关用户权限**。因此，在配置文件前，需要先通过 Web 界面开启这两项开关。

- **开启系统 SSH 服务**：登录 fnOS Web 管理界面，进入 `系统设置` -> `SSH`，将服务开关打开。
- **授权用户 SSH 登录**：同样在 `系统设置` -> `用户管理` 中，找到你要用来登录的用户（如 `admin`），点击“更多”并**启用 SSH**。

完成这两步后，就可以使用密码通过 SSH 客户端登录系统了，这是进行后续配置的前提。

#### 2. 修改 SSH 服务配置
这一步的核心是修改 SSH 的配置文件 `sshd_config`，以禁用密码登录。

- **登录终端**：使用 SSH 客户端（如 `终端`、`PuTTY`）或飞牛应用中心的 `FntermX 终端` 应用，通过管理员账号和密码登录系统。

- **编辑配置文件**：在终端中执行以下命令：
   ```bash
   sudo nano /etc/ssh/sshd_config
   ```

- **修改参数**：在文件中找到并修改以下参数。如果参数前有 `#`，请删掉它来取消注释。
   ```bash
   # 将 yes 改为 no，禁止使用密码登录
   PasswordAuthentication no
   
   # 建议保持为 no，禁止 root 用户直接登录以提升安全性
   PermitRootLogin no
   ```

- **重启服务**：修改完成后，按 `Ctrl + X`，再按 `Y` 和 `Enter` 保存并退出。然后执行以下命令使配置生效：
   ```bash
   sudo systemctl restart ssh
   ```

#### 3. 设置 SSH 密钥登录（重要）
仅关闭密码登录还不够，你必须在关闭密码**之前**配置好密钥，否则你自己也将无法登录。

- **生成密钥对（在本地电脑操作）**：
  打开你本地电脑的终端（Windows 可用 PowerShell 或 CMD），执行：
  ```bash
  ssh-keygen -t ed25519 -C "your_email@example.com"
  ```
  按提示操作。为更安全，可以在提示 `Enter passphrase` 时设置一个密钥密码。

- **上传公钥至 fnOS**：
  该命令可将公钥一键上传。请将 `用户名` 和 `服务器IP` 替换为你的实际信息。
  ```bash
  ssh-copy-id 用户名@你的飞牛IP地址
  ```
  > **如果没有 `ssh-copy-id` 命令（如 Windows），可以用 `nano` 命令在飞牛上手动创建：**
  > 1.  在飞牛终端中执行 `nano ~/.ssh/authorized_keys`。
  > 2.  将本地生成的公钥（`.pub` 文件里的内容）完整复制进去，`Ctrl+X` 保存退出。

- **测试登录**：
  完成上传后，退出当前 SSH 会话，尝试重新登录。如果无需输入密码即可登录，则代表密钥配置成功。

### ⚠️ 关键提醒

1.  **严格遵守操作顺序**：在关闭密码登录前，请务必确保：① SSH 密钥已成功配置；② 能用密钥成功登录系统。否则一旦关闭密码，你可能会被锁在系统外。
2.  **妥善备份私钥**：你电脑上的私钥文件（默认在 `~/.ssh/id_ed25519`，没有 `.pub` 后缀）是唯一凭证，一旦丢失，你将无法登录系统，请务必妥善保管。
